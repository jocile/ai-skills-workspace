# TASKS: Portal de Documentação e Catálogo de Skills

**Status:** Pronto para Execução

**Especificação Principal:** `SPEC.md`

**Arquitetura:** `PLAN.md`

Este documento mapeia as unidades de trabalho para a implementação da infraestrutura estática baseada em Docusaurus, isolamento de rotas e o pipeline de integração contínua.

---

## Milestone 1: Fundação da Infraestrutura (Setup & Core)

**Objetivo:** Estabelecer o ecossistema de desenvolvimento e configurar o isolamento de domínios.

* [ ] **1.1 Inicializar Projeto Base**
* **Descrição:** Gerar a estrutura inicial utilizando o template classic do Docusaurus (v3+). Assegurar a estabilidade do setup e dependências para execução via Node.js v20 em ambiente host Windows 11 e versionar no Git.

* **Critérios de Aceite (DoD):**
* Servidor local executando sem falhas em `localhost:3000` via comando `npm run start`.

* [ ] **1.2 Isolar Instâncias de Documentação (`docusaurus.config.js`)**
* **Descrição:** Parametrizar o `docusaurus.config.js` para gerenciar a instância default (rotas de `/docs/`) e instanciar o `@docusaurus/plugin-content-docs` para a rota isolada `/skills/`.

* **Critérios de Aceite (DoD):**
* Configuração de Múltiplas Instâncias válida e livre de colisões.

* [ ] **1.3 Segmentar Árvore de Navegação**
* **Descrição:** Separar o controle das sidebars para evitar renderização cruzada nos menus visuais.

* **Critérios de Aceite (DoD):**
* `sidebars.js` mantido para documentação conceitual.

* `sidebarsSkills.js` criado operando de forma autogerada (autogenerated) a partir da raiz da pasta `skills/`.

## Milestone 2: Engenharia de Dados (Contrato Híbrido)

**Objetivo:** Implementar a arquitetura de arquivos e templates para que os metadados consumidos pelo Antigravity e pelo React sejam perfeitos.

* [ ] **2.1 Desenvolver Template Ouro (`_template.md`)**
* **Descrição:** Criar o arquivo de referência que será utilizado para instanciar novas habilidades no repositório.

* **Critérios de Aceite (DoD):**
* Arquivo salvo em `skills/_template.md`.

* Front-matter contém explicitamente o schema estrito com as chaves `name`, `title` e `description` vazias para preenchimento.

* [ ] **2.2 Popular Catálogo Base**
* **Descrição:** Cadastrar markdowns iniciais para teste do frontend utilizando os parâmetros exigidos.

* **Critérios de Aceite (DoD):**
* Arquivo `skills/analisador-logs.md` e `skills/readme-generator.md` redigidos e em conformidade estrutural plena com a chave híbrida.

## Milestone 3: Interface Dinâmica (React & MDX)

**Objetivo:** Elaborar o catálogo visual isolado para exibir os metadados extraídos da instância das skills.

* [ ] **3.1 Encapsular Estilização do Catálogo**
* **Descrição:** Criar o arquivo `styles.module.css` focado na renderização em grid das habilidades, mapeando as variáveis nativas de tema do Docusaurus.

* **Critérios de Aceite (DoD):**
* Comportamento de transição nos cards (hover) e layout responsivo aplicados corretamente na rota web.

* [ ] **3.2 Implementar Lógica do Componente `<SkillCards/>**`
* **Descrição:** Consumir de forma limpa a API `usePluginData` restrita ao ID da instância `skills` para iterar e gerar os cartões React.

* **Critérios de Aceite (DoD):**
* Os metadados `title` e `description` são extraídos e renderizados na interface sem retornos nulos ou em branco.

* O Call to Action (CTA) do card redireciona para a rota correta do artefato estático (ex: `/skills/analisador-logs`).

---

## Milestone 4: Governança e CI/CD (Fail-Fast)

**Objetivo:** Automação de segurança para garantir a aderência imutável ao `SPEC.md` através das actions no repósitorio.

* [ ] **4.1 Desenvolver Linter do Contrato de Dados**
* **Descrição:** Escrever o processo de checagem pré-build que faz a leitura dos blocos YAML na pasta `/skills/`.

* **Critérios de Aceite (DoD):**
* Script configurado como `npm run validate-skills`.

* O processo emite o código `exit 1` interrompendo o fluxo imediatamente caso um arquivo da pasta falhe ao declarar `name`, `title` ou `description`.

* [ ] **4.2 Orquestrar Pipeline de Entrega (`deploy.yml`)**
* **Descrição:** Estabelecer o workflow de deploy e testes de sanidade contínua no GitHub Actions.

* **Critérios de Aceite (DoD):**
* Pipeline reage ao trigger de *Push* na branch `main`.

* Workflow executa a validação do contrato (linter do passo anterior), compilação via `npm run build` (checando links quebrados nativamente) e, em caso de sucesso, efetua o deploy automático.
